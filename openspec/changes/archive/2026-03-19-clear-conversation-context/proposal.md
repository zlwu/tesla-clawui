## Why

当前聊天会话会持续累积历史消息，用户在切换话题、纠正模型偏航或准备开始一轮全新对话时，缺少一个明确、可控的方式清除之前的上下文。这会让后续回复继续受旧消息影响，也让 Tesla 车机端的最近消息恢复与显示语义变得不够直接，因此需要提供一个轻量且可预期的“清除上下文”能力。

## What Changes

- 在 Tesla 聊天主界面顶栏右侧通过一个折叠的 `...` 菜单承载“清除上下文”入口，用于清空当前会话下已持久化的消息历史。
- 用户执行清除后，当前消息列表立即回到空状态，后续发送的第一条消息必须在不带旧历史上下文的前提下开始新的对话。
- 清除成功后保持当前本地 app session 不变，但需要轮换上游 OpenClaw session key，避免继续命中旧的 gateway 会话上下文。
- 清除完成后，页面刷新或重新进入会话时，不再恢复被清除的旧消息。
- 为给菜单入口腾出空间，当前顶栏中的多个文字状态提示将收敛为一个更紧凑的无边框状态指示器，优先使用独立色点或小图标，而不是并排堆叠多个文字 pill 或按钮壳体。
- 清除动作不扩展为多会话管理、历史归档、软删除回收站或手机端协同，只覆盖当前共享 PIN 解锁后的单会话主路径。
- 为清除成功、清除失败、流式回复进行中不可清除等关键路径补充验证与测试，避免破坏现有 SSE 文本主链路。

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `session-and-message-lifecycle`: 新增清除当前会话消息历史并在后续恢复中保持空状态的要求。
- `tesla-chat-ui`: 新增 Tesla 主界面上的清除上下文入口、直接执行语义和清除后的界面反馈要求。
- `text-chat-streaming`: 明确清除上下文后后续文本请求必须基于已清空的会话历史处理，且流式进行中需要受控处理清除动作。

## Impact

- 前端主界面状态与 DOM：新增清除入口、禁用条件、OpenClaw session key 轮换和清除后的空态刷新。
- 后端会话/消息服务与 API：需要提供按当前 session 清除消息历史的安全接口，并保证后续文本请求不再读取已清除消息。
- 数据层：需要删除或失效化当前 session 下的正式消息记录，并处理与请求幂等结果相关的残留状态。
- 测试与验证：需要补充前后端单元测试、集成测试以及针对 Tesla 主路径的手动验证步骤。
